# 小林 健太

## 担当領域
インフラ・CI/CD全般。AWS CDKによるインフラ構築（Lambda, DynamoDB, S3, CloudFront, API Gateway）、GitHub ActionsによるCI/CDパイプライン、環境管理（dev/stg/prod）を担当。監視・ログ基盤（CloudWatch, X-Ray）の設定も担当領域。

## ひととなり
フトアゴヒゲトカゲの「ゴジラ」を飼っている。チェックインで爬虫類の話になると急に詳しくなる。基本的に寡黙だが、インフラやAWSの話題では的確な発言をする。「無人島に持っていくもの」では少し考えてから「ナイフ」と一言。実用主義。コーヒーにこだわりがあり、自分で豆を挽いているらしい。

## 仕事の傾向
手が早い。タスクを振られると黙々と進めて、気づいたら完了している。CDKのコードは整理されていて、他のメンバーが見ても分かりやすい構成にしている。問題が起きた時の切り分けも速い。一方で、自分から進捗を報告するのが少ないので、渡辺さんが朝会で「小林さん、どうですか？」と振る場面が多い。

## コミュニケーションの特徴
発言は短いが的確。「できます」「やっておきます」「問題ないです」が頻出ワード。Slackでの報告も箇条書きで簡潔。ただしインフラ構成の説明を求められた時は図を描いて丁寧に説明する。木村さんとの技術議論ではテンポよくやり取りする。中村さんの質問には一言で答えがちだが、悪気はなく、聞かれれば追加説明する。

## 顧客対応メモ
企画部との会議ではほぼ発言しない。インフラの話題を振られた時だけ簡潔に回答する。「デプロイは自動化されてます」「SLAは99.9%を目標にしてます」など、聞かれたことにだけ答えるスタイル。チーム内共有では「企画部の人はインフラの話は興味ないから、聞かれなければ言わなくていい」と割り切っている。

## コンディション推移
スプリント2、スプリント3を通じて安定している。コンディションの波が小さいタイプ。唯一、3/19にCI/CDパイプラインのGitHub Actionsが壊れた時だけ「ちょっと厄介です」と言っていたが、翌日には修正完了。3/25のリファインメントではスプリント4のインフラタスクが少なめと分かり、少しリラックスした様子。
