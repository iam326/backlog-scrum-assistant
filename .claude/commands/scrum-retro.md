ふりかえりの文字起こしを処理します。議事録保存、KPT整理、メンバープロフィール更新を行います。

$ARGUMENTS

1. 文字起こしを取得する:
   - 引数でファイルパスが渡された場合はそれを使う
   - なければ `data/transcripts/今日の日付/retro.txt` を確認し、あれば使う
   - どちらもなければユーザーにファイルの場所を聞く

2. 議事録として保存:
```
python3 cli.py save-daily retro -f [ファイルパス]
```

3. 「miroのスクショはありますか？」とユーザーに確認。あれば画像を読み取り、KPTの内容を抽出。

4. 出力:

### ふりかえりサマリ
- KPTの内容を整理（文字起こし + miroスクショから統合）

### アクションアイテム
- 誰が何をいつまでにやるか

### 未解決のトピック
- 議論しきれなかったもの → 次回の議題候補として記録

### メンバーの発言傾向
- 誰がどんなK/P/Tを出したかの傾向分析
- 「Pが多い人 → 不満を感じている可能性」「Kを出さない人 → 承認が足りてない可能性」等の洞察

5. メンバープロフィール更新（`data/members/[名前].md`）:

ふりかえりは特に人間性が出る場なので、以下を重点的に更新:
- **ひととなり**: 価値観、仕事への姿勢、チームに対する思い
- **仕事の傾向**: 改善意識の高さ、問題の捉え方（技術寄り/プロセス寄り/人間関係寄り）
- **コミュニケーションの特徴**: 不満の表現の仕方、建設的か否か、他人のKPTへの反応
- **コンディション推移**: 前週と比べて発言のトーンに変化があるか
- **記録セクション**: KPTの内容と発言要約を追記

過去のふりかえり記録と比較し、繰り返し出ているProblemや、前回のTryが実行されたかどうかも確認・コメントしてください。

6. 上記の出力内容を `data/minutes/YYYY-MM-DD/retro.md`（今日の日付）に保存してください。同じ日に再実行した場合は上書きしてください。
