朝会の文字起こしを処理します。議事録保存、Backlog差分検出、メンバープロフィール更新を一括で行います。

朝会は2部構成です:
- **朝会（顧客・他ベンダー同席）**: チェックイン → 作業報告
- **チーム内共有（自社メンバーのみ）**: チーム内相談、顧客前では言えない話

$ARGUMENTS

1. 文字起こしを取得する:
   - 引数でファイルパスが渡された場合はそれを使う
   - なければ `data/transcripts/今日の日付/` を確認し、`standup.txt` や `team.txt` があれば使う
   - どちらもなければユーザーにファイルの場所を聞く
   - 朝会とチーム内共有が1つのテキストに入っている場合は区切りを判別する

2. 議事録を保存:
朝会:
```
python3 cli.py save-daily standup -f [朝会ファイル]
```
チーム内共有がある場合:
```
python3 cli.py save-daily team -f [チーム内共有ファイル]
```

3. Backlogとの差分分析を実行:
```
python3 cli.py sync -f [朝会ファイル]
```

4. 出力（以下の順で）:

### チェックイン回答
メンバーごとにお題と回答を整理

### 作業報告 vs Backlog差分
- 口頭で報告されたがBacklogに反映されていない作業
- Backlogでは進行中/Blockedだが報告に出てこなかった課題
- ステータスの食い違い
- → **「Backlog更新してね」リストを生成**

### チーム内共有サマリ（ある場合）
- 議論されたトピックと結論
- アクションアイテム（誰が何をいつまでに）
- 顧客に関する懸念事項や対応方針（**これは特に重要 — 顧客対応メモとしてプロフィールにも記録**）

### 気づき
- 全体を通して気になった点

5. メンバープロフィール更新（`data/members/[名前].md`）:

ファイルがなければ `data/members/_template.md` を元に新規作成。

以下の観点で更新:
- **ひととなり**: チェックイン回答から見える趣味・価値観
- **仕事の傾向**: 報告の仕方、タスクの進め方、Backlog更新の習慣
- **コミュニケーションの特徴**: 議論スタイル、質問傾向、ブロッカーの伝え方
- **顧客対応メモ**: 朝会での顧客とのやり取り + チーム内共有で出た顧客に関する話題（懸念、対応方針、気にしているポイント等）
- **コンディション推移**: 発言量やトーンに普段と違いがあれば記録
- **記録セクション**: 日付と要約を追記

過去の記録と比較して変化や傾向があれば明示的にコメントしてください。
