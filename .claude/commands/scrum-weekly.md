リファインメントの文字起こしを処理します。議事録保存、スプリントサマリ、メンバープロフィール更新を一括で行います。

$ARGUMENTS

1. 文字起こしテキスト（上記 or 入力を求める）を受け取る

2. 文字起こしを一時ファイルに保存し、議事録として保存:
```
python3 cli.py save-daily refinement -f [一時ファイルパス]
```

3. Backlogデータを取得:
```
python3 cli.py weekly
```

4. Backlog差分分析:
```
python3 cli.py sync -f [一時ファイルパス]
```

5. 文字起こしの構成を把握してください。リファインメントは以下の流れです:
- **チェックイン**: お題に対する各メンバーの回答
- **全体議論**: チーム全体で議論すべきトピック
- **作業報告**: 各メンバーの進捗
- **次スプリント計画**: 次にやることの話

6. 出力（以下の順で）:

### チェックイン回答
メンバーごとにお題と回答を整理

### スプリントサマリ
- 完了/進行中/残の概況

### 全体議論の要約
- 議論されたトピックと結論・アクションアイテム

### 作業報告 vs Backlog差分
- scrum-daily と同じ形式で差分を出力

### 次スプリントの計画
- 議論された内容から次スプリントでやることを整理

### 気になるシグナル
- weeklyコマンドで検出されたシグナルの深掘り

### 次回リファインメントの議題候補
- 今回議論しきれなかった点 + Backlogから検出された議題候補

7. メンバープロフィール更新（`data/members/[名前].md`）:

scrum-dailyと同じ観点に加え:
- **仕事の傾向**: 見積もりの精度、タスク分解の仕方、技術的な議論の深さ
- **コミュニケーションの特徴**: リファインメントでの発言傾向（質問が多い/仕様を掘り下げる/黙っている等）
- **顧客対応メモ**: 顧客要件に関する発言や懸念点
